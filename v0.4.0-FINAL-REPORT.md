## 🎯 Мануальная активация (User Control)

### Философия:
- По умолчанию: **Linear search** (точный, простой, работает как v0.3.3)
- Опционально: **HNSW search** (быстрый, approximate, user-activated)

### Процесс активации:
```bash
# Шаг 1: Проверить статус
npm run search:info

# Шаг 2: Если граф >1000 nodes, активировать HNSW
npm run search:enable-hnsw

# Шаг 3: Benchmark (опционально)
npm run search:benchmark

# Шаг 4: Использовать (автоматически использует HNSW)
npm run query "python web framework"
```

### Преимущества подхода:
- ✅ **User control** - пользователь решает когда включать
- ✅ **No breaking changes** - старый код работает без изменений
- ✅ **Reversible** - можно вернуться на linear (npm run search:disable-hnsw)
- ✅ **Transparent** - ясный процесс с feedback
- ✅ **Progressive enhancement** - система растёт с потребностями

## ⚠️ Известные ограничения

### 1. HNSW требует интернет
**Причина:** Embeddings через Gemini API
**Workaround:** Use linear search offline
**Fix:** v0.4.2 - offline mode

### 2. Build time для больших графов
**Проблема:** 1000 nodes: 30-40s, 5000 nodes: 2-3 min
**Workaround:** Build once, reuse many times
**Fix:** v0.4.1 - incremental updates

### 3. Approximate search (95-99% vs 100%)
**Проблема:** Trade-off speed vs accuracy
**Workaround:** Увеличь efSearch для точности
**Fix:** Это inherent trade-off HNSW, не bug

### 4. No auto-rebuild после consolidation
**Проблема:** Требуется ручной rebuild
**Workaround:** npm run search:rebuild-hnsw
**Fix:** v0.4.1 - auto-rebuild trigger

## 🎓 Технологии

### Vectra
- Pure JavaScript HNSW
- File-based storage
- No native dependencies
- Works on Windows/Mac/Linux
- GitHub: https://github.com/Stevenic/vectra

### HNSW Algorithm
- Yu. A. Malkov, D. A. Yashunin (2016)
- Paper: https://arxiv.org/abs/1603.09320
- Logarithmic complexity: O(log N)
- 100-1000x faster than linear

### Gemini API
- Embeddings: 3072-dim vectors
- Model: text-embedding-004
- Cost: $0.50/месяц (кэшированные)
