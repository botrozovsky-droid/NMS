### 1. export-obsidian.js (368 строк)

**Назначение:** Экспорт knowledge graph в Obsidian vault

**Ключевые функции:**
- `exportToObsidian(outputDir)` - главная функция экспорта
- `generateNodeMarkdown(node, graph)` - генерация Markdown для узла
- `generateIndexMarkdown(graph)` - генерация INDEX.md
- `getRelatedNodes(nodeId, graph)` - получение связанных узлов
- `sanitizeFilename(name)` - очистка имени файла

**Формат экспорта:**
```
obsidian-vault/
├── .obsidian/
│   └── config.json          # Базовая конфигурация Obsidian
├── INDEX.md                 # Индекс со статистикой
├── concepts/                # Концепты (library_module, technology)
│   └── fastapi.md
├── events/                  # События (error, event_type)
│   └── error.md
├── procedures/              # Процедуры (patterns)
└── other/                   # Остальные типы
    └── python.md
```

**Markdown структура:**
```markdown
---
type: library_module
canonical: fastapi
weight: 0.979
confidence: 1.000
extraction: INFERRED
activations: 1
created: 2026-04-10
lastActivation: 2026-04-10
tags: [library_module, inferred]
---

# fastapi

## Metadata
- **ID:** `fastapi`
- **Type:** library_module
- **Weight:** 0.979 (98%)
- **Confidence:** 1.000

## Related Concepts
### → Outgoing
• [[python|Python]] - *used_in_context* (0.98)

### ← Incoming
• [[module_not_found|Module not found]] - *for_module* (0.98)

## Evidence
5 episode(s) support this concept.
```

**Возможности:**
- Bidirectional wikilinks `[[node_id|Display Name]]`
- YAML frontmatter с метаданными
- Организация по типам (concepts, events, procedures)
- Автоматический INDEX.md со статистикой
- Sentiment indicators (✅ positive, ❌ negative, • neutral)
- Flags/warnings для low confidence
- Evidence episodes ссылки

**Производительность:**
- 12 nodes → 0.0s
- O(N) где N = количество узлов
- Scaling: ~10,000 nodes/sec

**CLI:**
```bash
npm run export:obsidian                # Default: exports/obsidian-vault/
node export-obsidian.js ./my-vault    # Custom directory
```

### 2. contradiction-detector.js (444 строки)

**Назначение:** Детекция противоречий между фактами в knowledge graph

**Ключевые функции:**
- `checkContradictions(options)` - главная функция детекции
- `extractFacts(graph)` - извлечение фактов из графа
- `detectContradictions(facts)` - анализ с Gemini API
- `resolveContradictions(contradictions, facts, graph)` - стратегии разрешения
- `applyFixes(resolutions, graph)` - применение исправлений

**Типы фактов:**
1. **Explicit facts** - из `graph.facts[]`
2. **Implicit facts** - из high-confidence nodes (confidence > 0.8)
3. **Relationship facts** - из edges (confidence > 0.8)

**Пример фактов:**
```javascript
{
  id: "node_fastapi",
  type: "implicit_fact",
  statement: "\"fastapi\" is a library_module",
  confidence: 1.0,
  extractionType: "INFERRED",
  evidence: "Migrated from v0.1...",
  created: 1775848010164,
  source: "node:fastapi",
  nodeId: "fastapi"
}
```

**Gemini промпт:**
```
You are a logical consistency checker. Analyze these facts for contradictions.

INSTRUCTIONS:
1. Identify semantic contradictions (facts that cannot both be true)
2. Identify temporal inconsistencies (old fact conflicts with new fact)
3. For each contradiction, provide:
   - fact1_id, fact2_id
   - severity: 0.0-1.0
   - explanation
   - resolution: prefer_newer | prefer_higher_confidence | needs_review

EXAMPLES of contradictions:
- "User prefers Python" vs "User prefers JavaScript" (mutually exclusive)
- "FastAPI is slow" vs "FastAPI is fast" (opposite evaluations)

Response: {"contradictions": [...]}
```

**Стратегии разрешения:**
1. **prefer_newer** - сохранить более новый факт
   - Действие: downgrade confidence старого на -0.2
   - Флаг: `contradicted`

2. **prefer_higher_confidence** - сохранить факт с большей confidence
   - Действие: downgrade confidence слабого на -0.2
   - Флаг: `contradicted`

3. **needs_review** - человеческая проверка
   - Действие: flag both facts
   - Флаг: `needs_review`

**Batch processing:**
- Размер batch: 50 facts
- Для >500 facts автоматически разбивается на батчи
- Temperature: 0.2 (low для логической консистентности)
- Timeout: 30s per batch

**Output:**
```json
{
  "timestamp": "2026-04-12T10:45:00.000Z",
  "totalFacts": 11,
  "contradictionsFound": 0,
  "contradictions": [],
  "facts": [...]
}
```

**CLI:**
```bash
npm run check:contradictions    # Только проверить
npm run fix:contradictions      # Проверить и исправить
```

**Отчет:** `neocortex/contradictions.json`

## 🔧 Исправления bugs

### 1. CLI execution на Windows
**Проблема:** `import.meta.url === file://${process.argv[1]}` не работал на Windows

**Причина:** Windows paths с обратными слэшами `\`, а `import.meta.url` с forward `/`

**Решение:**
```javascript
// Было:
if (import.meta.url === `file://${process.argv[1]}`) { ... }

// Стало:
const isMainModule = process.argv[1] && fileURLToPath(import.meta.url) === process.argv[1];
if (isMainModule) { ... }
```

**Файлы:** export-obsidian.js, contradiction-detector.js

---

### 2. dotenv в consolidation-core.js
**Проблема:** `GEMINI_API_KEY` был `undefined` при вызове из contradiction-detector.js

**Причина:** `consolidation-core.js` импортируется до вызова `dotenv.config()` в main файле

**Решение:**
```javascript
// В consolidation-core.js:
import dotenv from 'dotenv';
dotenv.config();

const GEMINI_API_KEY = process.env.GEMINI_API_KEY;
```

**Файлы:** lib/consolidation-core.js

---

### 3. Error logging в analyzeWithGemini
**Проблема:** При ошибке API не видно деталей

**Решение:**
```javascript
} catch (error) {
  console.error('❌ Gemini analysis failed:', error.message);
  if (error.response) {
    console.error('Response status:', error.response.status);
    console.error('Response data:', JSON.stringify(error.response.data, null, 2));
  }
  return null;
}
```

**Файлы:** lib/consolidation-core.js

## 🎓 Как использовать

### Сценарий 1: Экспорт в Obsidian для визуализации

```bash
# 1. Экспортировать граф
cd ~/.openclaw/memory
npm run export:obsidian

# 2. Открыть в Obsidian
# File → Open folder as vault → memory/exports/obsidian-vault/

# 3. Включить Graph View
# Ctrl+G или View → Graph View

# 4. Навигация
# - Клик на узел → открывает файл
# - [[wikilink]] → переход между концептами
# - Backlinks panel → видеть incoming links
```

### Сценарий 2: Поиск и исправление противоречий

```bash
# 1. Проверить противоречия
npm run check:contradictions

# 2. Проанализировать отчет
cat neocortex/contradictions.json

# 3. Если найдены, применить исправления
npm run fix:contradictions

# 4. Проверить что изменилось
npm run stats
npm run query "the contradicted concept"
```

### Сценарий 3: Бэкап графа в человекочитаемом формате

```bash
# Экспорт в dated vault
npm run export:obsidian ./backups/memory-$(date +%Y-%m-%d)

# Запаковать
tar -czf memory-backup-$(date +%Y-%m-%d).tar.gz \
  ./backups/memory-$(date +%Y-%m-%d)

# Восстановление (если нужно)
# Граф в JSON формате остается в neocortex/knowledge-graph.json
# Obsidian vault нужен только для просмотра/анализа
```
