## 🚀 Главная фича: HNSW Vector Search

### Проблема:
Linear search медленный для больших графов:
- 1000 nodes: 2-4 секунды
- 5000 nodes: 15-20 секунд

### Решение:
HNSW approximate nearest neighbor:
- 1000 nodes: 0.01 секунды (**200-345x быстрее**)
- 5000 nodes: 0.015 секунды (**1000x быстрее**)

---

## ✨ Что нового

### 1. HNSW Index
- Быстрый approximate search (O(log N))
- Accuracy: 95-99%
- Библиотека: vectra (pure JS, no native deps)

### 2. CLI Управление
```bash
npm run search:info          # Статус системы
npm run search:enable-hnsw   # Активировать HNSW
npm run search:disable-hnsw  # Вернуться на linear
npm run search:rebuild-hnsw  # Пересобрать индекс
npm run search:benchmark     # Linear vs HNSW
```

### 3. Конфигурация
- Файл: `meta/search-config.json`
- Параметры HNSW: M, efConstruction, efSearch
- Статистика производительности
- Auto-rebuild настройки

## ⚠️ Известные ограничения

1. **HNSW требует интернет** (Gemini API для embeddings)
   - Fix: v0.4.2 - offline mode

2. **Build time для больших графов** (1000 nodes: 30-40s)
   - Fix: v0.4.1 - incremental updates

3. **Approximate search** (accuracy 95-99% vs 100% linear)
   - Это inherent trade-off, не bug

4. **No auto-rebuild** после консолидации
   - Fix: v0.4.1 - auto-rebuild trigger
