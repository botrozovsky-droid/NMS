# OpenClaw Memory System - Руководство пользователя 🧠

## Что это такое?

Представьте, что у AI (искусственного интеллекта) есть **собственный мозг с памятью**, который работает почти как человеческий. Наша система - это и есть этот "мозг".

---

## 🧠 Как работает человеческая память (для сравнения)

Когда вы общаетесь с кем-то, ваш мозг делает несколько вещей:

1. **Сенсорная память** - вы слышите/видите информацию прямо сейчас
2. **Кратковременная память** - вы помните разговор последних минут
3. **Долговременная память** - важные вещи откладываются навсегда
4. **Забывание** - неважное постепенно стирается

### Например:
- Вы встретили друга ☕
- Он упомянул, что любит пиццу с ананасами 🍕
- Мозг решает: "Это важно!" (потому что вы дружите)
- Информация сохраняется в долговременной памяти
- Через месяц вы помните и предлагаете ему пиццу с ананасами!

## ✨ Наше решение: OpenClaw Memory

Мы создали систему, которая работает **как настоящий человеческий мозг**:

### 📚 Аналогия с библиотекой

Представьте огромную библиотеку:

1. **Сенсорный буфер** = Стол у входа
   - Сюда кладут все новые книги (события)
   - Временное хранение

2. **Гиппокампус** = Комната сортировки
   - Библиотекарь смотрит: "Это важно или нет?"
   - Важные книги отмечаются стикером ⭐

3. **Неокортекс** = Главный зал библиотеки
   - Здесь хранятся все важные знания
   - Организовано как **граф связей** (об этом ниже)

4. **Процедурная память** = Кабинет "Как делать"
   - Здесь записаны привычки и предпочтения
   - "Пользователь предпочитает Python, а не JavaScript"

## 🕸️ Граф знаний - главная идея

### Старый подход (простой список):

```
📋 Список знаний:
- Python
- FastAPI
- Веб-разработка
- Пользователь любит Python
```

**Проблема:** Как связаны эти вещи? Непонятно!

### Наш подход (граф связей):

Представьте **карту метро**, где:
- **Станции** = понятия (Python, FastAPI, веб-разработка)
- **Линии между станциями** = связи (FastAPI использует Python)
- **Толщина линий** = насколько сильная связь

```
       Пользователь
           ↓ (любит)
        Python ←----→ FastAPI
           ↓           ↓
      (используется) (это)
           ↓           ↓
     Веб-разработка
```

**Преимущество:** AI видит не просто список, а **понимает связи**!

## 🌙 Как информация попадает в долговременную память?

### Процесс "Консолидация" (как во сне)

Когда вы спите, ваш мозг:
1. Прокручивает события дня
2. Отбирает важное
3. Укрепляет связи между воспоминаниями
4. Стирает ненужное

**Наша система делает то же самое!**

### Раньше (до v0.3):
- ⏰ Консолидация раз в сутки (ночью)
- ⏳ Новая информация доступна через 24 часа
- Проблема: "AI забывает" в течение дня

### Теперь (v0.3):
- ⚡ Мини-консолидация в конце каждой сессии (30 минут неактивности)
- ⏱️ Новая информация доступна через 5 минут!
- ✅ AI помнит важное сразу

### Аналогия:
**Раньше:** Как студент, который учит всё перед экзаменом раз в месяц
**Теперь:** Как студент, который повторяет материал после каждой лекции

### 2️⃣ Транзакции (защита от ошибок)

**Проблема:** Что если во время сохранения памяти произошла ошибка?

#### 🏦 Аналогия с банком:

**Без транзакций (опасно):**
1. Снять 1000₽ со счёта А ✅
2. **ОШИБКА! Электричество отключилось** ⚡
3. Положить 1000₽ на счёт Б ❌ (не успело)
4. Результат: **деньги пропали!** 💸

**С транзакциями (безопасно):**
1. Банк делает копию всех данных 📋
2. Снять 1000₽ со счёта А
3. **ОШИБКА!** ⚡
4. Банк автоматически **восстанавливает** всё из копии ↩️
5. Результат: **ничего не потеряно!** ✅

**Наша система теперь делает так же:**
- Перед любым изменением памяти → создаётся копия
- Если всё OK → копия удаляется
- Если ошибка → память восстанавливается из копии
- **Память никогда не повреждается!**

### 3️⃣ Быстрая память (консолидация в конце сессии)

#### 📝 Аналогия с дневником:

- Проблема: **Неделю не можете найти нужную заметку!**
